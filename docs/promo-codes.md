# Промокоды — API для витрины

Документ описывает, как фронтенд витрины взаимодействует с промокодами при оформлении заказа.

---

## 1. Валидация промокода

Перед оформлением заказа покупатель вводит промокод. Фронт отправляет его на проверку:

### `POST /promo-codes/validate`

**Request body:**

```json
{
  "code": "OLEG10"
}
```

| Поле   | Тип      | Обязательное | Описание                    |
| ------ | -------- | ------------ | --------------------------- |
| `code` | `string` | ✅           | Промокод (регистр не важен) |

**Ответ 200 (промокод валиден):**

```json
{
  "valid": true,
  "code": "OLEG10",
  "name": "Промо Олега",
  "type": "percent",
  "value": 10
}
```

| Поле    | Тип                      | Описание                                                        |
| ------- | ------------------------ | --------------------------------------------------------------- |
| `valid` | `boolean`                | Всегда `true` при 200                                           |
| `code`  | `string`                 | Нормализованный код (uppercase)                                 |
| `name`  | `string`                 | Человекочитаемое название промокода                             |
| `type`  | `"percent"` \| `"fixed"` | Тип скидки: процент или фиксированная сумма                     |
| `value` | `number`                 | Значение скидки (10 = 10% для percent, 100 = 100 грн для fixed) |

**Ответ 400 (ошибка):**

```json
{
  "statusCode": 400,
  "message": "Promo code not found"
}
```

Возможные сообщения ошибок:

- `Promo code is required` — пустой код
- `Promo code not found` — код не существует
- `Promo code is not active` — деактивирован админом
- `Promo code is not yet active` — ещё не наступила дата начала
- `Promo code has expired` — дата окончания прошла
- `Promo code usage limit reached` — исчерпан лимит использований

---

## 2. Применение промокода при оформлении заказа

Промокод передаётся как необязательное поле в теле запроса создания заказа.

### `POST /orders`

**Request body (фрагмент с промокодом):**

```json
{
  "phone": "+380671234567",
  "clientId": "abc-123-uuid",
  "items": [
    {
      "productId": "6650ab...",
      "sku": "SKU-001",
      "quantity": 2,
      "price": 500
    }
  ],
  "name": "Олег",
  "comment": "Доставка Новою Поштою",
  "deliveryFee": 80,
  "promoCode": "OLEG10"
}
```

| Поле        | Тип      | Обязательное | Макс. длина | Описание                |
| ----------- | -------- | ------------ | ----------- | ----------------------- |
| `promoCode` | `string` | ❌           | 50          | Промокод для применения |

> Все остальные поля (`phone`, `clientId`, `items`, `name`, `comment`, `deliveryFee`) без изменений — см. основную документацию заказов.

---

## 3. Что происходит на бэкенде

1. Бэк создаёт снепшот каждого товара и применяет **автоматические скидки** (Discounts).
2. Если передан `promoCode`:
   - Валидирует промокод (активен, даты, лимит).
   - Для каждого товара проверяет таргетинг (allowed/excluded products + categories).
   - Применяет скидку промокода **поверх** финальной цены после автоматических скидок (`priceFinal`).
   - Инкрементирует счетчик использований.
3. Если промокод невалиден или просрочен — заказ **не создаётся**, возвращается `400`.

---

## 4. Ответ заказа (поля связанные с промокодом)

В ответе `POST /orders` и `GET /orders/:id` появляются дополнительные поля:

### На уровне заказа

| Поле                | Тип              | Описание                                     |
| ------------------- | ---------------- | -------------------------------------------- |
| `promoCode`         | `string \| null` | Код промокода (если был применён)            |
| `promoCodeName`     | `string \| null` | Название промокода                           |
| `promoCodeDiscount` | `number`         | Общая сумма скидки по промокоду (все товары) |

### На уровне каждого item в `items[]`

| Поле            | Тип      | Описание                                                       |
| --------------- | -------- | -------------------------------------------------------------- |
| `promoDiscount` | `number` | Сумма скидки промокода для этого товара (0 если не применялся) |

## 5. Рекомендуемый UX-флоу

```
┌──────────────────────────────────────────────┐
│  Корзина                                     │
│                                              │
│  [Товар 1]  500 грн × 2 = 1000 грн          │
│  [Товар 2]  200 грн × 1 =  200 грн          │
│                                              │
│  ┌─────────────────────┬──────────┐          │
│  │ Введіть промокод    │ Перевірити│          │
│  └─────────────────────┴──────────┘          │
│                                              │
│  ✅ Промо Олега: -10%                        │
│                                              │
│  Доставка:              80 грн               │
│  Итого:               1160 грн               │
│                                              │
│  [Оформити замовлення]                       │
└──────────────────────────────────────────────┘
```

### Шаги:

1. Покупатель вводит промокод в инпут.
2. По нажатию «Перевірити» → `POST /promo-codes/validate`.
3. При `200` — показать название и тип/значение скидки (например «Промо Олега: -10%»).
4. При `400` — показать ошибку из `message` (можно маппить на свои переводы).
5. При оформлении заказа — передать `promoCode` в `POST /orders`.
6. Если бэк вернул `400` при создании заказа (например, лимит исчерпан между валидацией и оформлением) — показать ошибку.

## 7. Примечания

- Промокод необязателен — заказ создаётся и без него.
- Код регистронезависим: `oleg10`, `Oleg10`, `OLEG10` — одно и то же.
- Промокод применяется **после** автоматических скидок, т.е. скидка считается от уже финальной цены.
- Таргетинг промокодов (какие товары/категории) настраивается в админке — фронту витрины об этом знать не нужно, бэк сам определяет, к каким товарам применить скидку.
- Если промокод применим только к части товаров в корзине — скидка будет рассчитана только для них, остальные товары получат `promoDiscount: 0`.
